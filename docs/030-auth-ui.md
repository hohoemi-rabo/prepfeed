# 030: 認証UI（ログインページ・ユーザーメニュー）

## 概要

ログインページの作成と、ヘッダーへの認証状態表示（ログイン/ログアウト、ユーザーアバター）を実装する。

## 背景

認証基盤（029）の上にUI層を構築し、ユーザーがログイン・ログアウトできるようにする。

## 要件

### ログインページ（`/auth/login`）

- [ ] 「Googleでログイン」ボタン
- [ ] PrepFeedのロゴ・説明文
- [ ] 有料機能の概要紹介（なぜログインが必要か）
- [ ] ログイン中のローディング表示
- [ ] エラー時のメッセージ表示

### ヘッダー更新

- [ ] 未認証時: 「ログイン」ボタン表示
- [ ] 認証済み時: ユーザーアバター + ドロップダウンメニュー
  - ダッシュボードリンク
  - ログアウトボタン
- [ ] モバイルメニューにも同様の認証状態表示

### 認証状態の取得

- [ ] カスタムフック `useAuth`（またはSupabaseのセッション監視）
- [ ] 認証状態のリアルタイム反映

## 受け入れ条件

- [ ] `/auth/login` でログインページが表示される
- [ ] Googleログイン後にダッシュボードにリダイレクトされる
- [ ] ヘッダーにユーザーアバターが表示される
- [ ] ドロップダウンからログアウトできる
- [ ] 未認証時はヘッダーに「ログイン」ボタンが表示される
- [ ] モバイルメニューでも認証UIが正しく動作する
